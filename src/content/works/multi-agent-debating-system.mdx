## Project Overview

Developed an innovative multi-agent AI system that simulates autonomous debates between AI agents. Each agent has distinct roles and capabilities, creating dynamic, intelligent discussions on various topics using local language models and custom knowledge retrieval.

## System Architecture

### Agent Roles
- **Affirmative Agent**: Presents arguments supporting the topic
- **Negative Agent**: Provides counterarguments and critiques
- **Judge Agent**: Evaluates arguments and provides scoring
- **Orchestrator Agent**: Manages debate flow and coordination

### Technical Components
- **Local LLMs**: Ollama integration for privacy-preserving inference
- **RAG Pipeline**: Custom retrieval system using ChromaDB
- **Knowledge Base**: PDF-based document processing
- **UI Framework**: Streamlit for dynamic, interactive interface
- **Concurrency**: Threading for parallel agent operations

## Key Features

- **Autonomous Interaction**: Agents operate independently without human intervention
- **Context-Aware Responses**: RAG pipeline provides relevant knowledge for informed arguments
- **Dynamic UI**: Real-time debate visualization with Streamlit
- **Modular Design**: Easy to add new agent types or modify existing ones
- **Local Processing**: Complete privacy with local LLM inference

## Technical Implementation

### Agent Communication
```python
class DebateAgent:
    def __init__(self, role, knowledge_base):
        self.role = role
        self.knowledge_base = knowledge_base
        self.llm = OllamaClient()
    
    def generate_response(self, context, previous_arguments):
        # Retrieve relevant knowledge
        relevant_docs = self.knowledge_base.search(context)
        
        # Generate response based on role and context
        response = self.llm.generate(
            prompt=self.format_prompt(context, relevant_docs),
            role=self.role
        )
        
        return response
```

### RAG Pipeline
- **Document Processing**: PDF parsing and chunking
- **Embedding Generation**: Vector representations of knowledge
- **Similarity Search**: ChromaDB for efficient retrieval
- **Context Assembly**: Dynamic context building for agents
